🤖 Hermes Agent 架構與建制報告

從零開始打造一個「會做事的 AI 團隊」
以大學生也能聽懂的方式說清楚
📅 2026 年 6 月
🎯 一、這是什麼？—— 用生活比喻來理解

想像你開了一家公司，你不可能一個人做所有事。你需要：

🧠 策略長 → 🔨 工程師 → 🔍 品管員 → 📢 行銷 → 🔧 維運

在真實公司裡，這些是不同的員工。但在 CJ哥的世界裡，這些都是 同一個 AI（我，小賀 = Hermes） 扮演的不同角色。

簡單來說：Hermes Agent 就是一個「多功能 AI 員工」。給它一個任務，它會自己判斷該用什麼角色、什麼工具、走什麼流程來完成。就像一個超級助理，什麼都會，但每個技能有專門的「說明書」（skill）。
🏗️ 二、架構長什麼樣？

Hermes 的架構分三層，像三明治一樣：

第一層

🤖 本體 Layer

就是「我」—— 一個大型語言模型（LLM），負責理解指令、做決定、跟 CJ哥 對話。住在 Hostinger VPS 上，24 小時待命。

第二層

🛠️ 工具 Layer

我手邊有的「工具」：讀寫檔案、跑Terminal指令、開瀏覽器爬網頁、操作 GitHub、發 Telegram 訊息、跑 Python 腳本⋯⋯就像一個員工桌上擺了各種軟體。

第三層

📋 技能 Layer

每個角色有專屬的「技能包」（skill），寫成 SKILL.md。例如「寫文章」有寫作技能、「寫程式」有 debug 技能。技能會持續累積更新，越用越強。

👥 主要角色陣容（10 位「員工」）

角色白話翻譯專門做的事
Orchestrator🧠 總指揮拆任務、分配工作、決定誰做什麼

注意：這 10 個角色不是 10 個 AI！是同一個 AI（我）根據任務需要，載入不同「角色說明書」來做事。就像一個人穿不同制服做不同工作。
🔧 三、怎麼從零開始建起來的？

建置順序就像蓋房子：先打地基、再蓋骨架、最後裝修。

第一階段：打底

🏗️ VPS + Docker + Traefik

租了一台 Hostinger VPS（Linux 伺服器），用 Docker 架設 Hermes Agent 容器，再用 Traefik 管理網路流量。就像先租辦公室、拉網路線。

第二階段：核心

🤖 安裝 Hermes Agent + 設定模型

裝好 Agent 本體，設定 AI 模型（從 Gemini 換成 Owl-alpha），設定 API Key。就像請了一個員工並給他鑰匙。

第三階段：技能武裝

📚 安裝 Skills（技能包）

從 GitHub 安裝各種技能：GitHub 操作、Python 除錯、瀏覽器操作、Email、影片生成⋯⋯就像給員工上課訓練。每次遇到新任務，就會去 GitHub 找對應的技能來自動學習。

第四階段：內容引擎

⚙️ 建立 Content Engine（內容工廠）

搭建了自動內容產線：知識爬取 → 分析 → 寫文章 → 產影片 → 上社群。包含 7 個定時任務（cron job），每天自動運作。就像蓋了一條工廠流水線。

第五階段：第二大腦

🧠 建立第二大腦專案

打造 CJ哥的「第二大腦」知識庫（Obsidian + GBrain），所有學到的東西都會整理進去，下次需要時自動查找。就像給 CJ哥 做了一個外部記憶裝置。

第六階段：多 Agent 協作

👥 建立 Swarm 架構

設定 10 個角色的協同工作模式（swarm.yaml），定義每個角色的工具、技能、工作規範。就像寫了一本「公司組織章程」。

🎛️ 四、Harness Engineering 是什麼？

這是整個系統最核心的概念，但也是最難懂的。讓我用比喻說清楚：

Harness（韁繩），就像騎馬時套在馬身上的那組裝備——不是馬本身，而是讓你能駕馭馬的東西。

為什麼需要 Harness？

AI 很聰明，但也很「衝動」。你叫它 Write code，它可能直接把你整個 repository 刪掉重寫。Harness 做的事情就是：

韁繩 #1

🔍 上下文注入

每次 AI 做事前，先自動讀取相關記憶、技能說明、過往經驗。就像員工上班前先看今天的工作備忘錄。不會每次都是「失憶狀態」重新開始。

韁繩 #2

🪜 技能載入

根據任務類型，自動載入對應的 SKILL.md。就像醫生看不同科別時，會翻不同的臨床指引。不會用做菜的步驟去修車。

韁繩 #3

🔄 Memory 管理

好的經驗自動存成記憶（memory tool），犯過的錯不會再犯。就像學生寫錯題本，大考前先看一遍。記憶有上限，所以越精簡越好。

韁繩 #4

✅ 安全守門員

任何「破壞性」操作（刪檔案、推程式碼、發對外訊息）都需要人工確認（greenlight）。就像銀行大額轉帳要雙重確認。AI 可以規劃但不能擅自行動。

韁繩 #5

📐 產出格式化

AI 的產出必須符合特定格式：統一的 commit message 格式、JSON 結構、報告範本。就像寫作文要照格式寫，不能隨便。

韁繩 #6

🛟 優雅降級

API key 過期？先存本地，標記待辦，通知 CJ哥。網路不通？重試三次，換備案。就像員工遇到問題不會停擺，而是先找替代方案。

總結一句話：Harness Engineering 就是「讓 AI 從『什麼都能聊』變成『什麼都能可靠地完成工作』」的工程方法。重點不在 AI 多聰明，而在於你給它套上的那組機制有多完善。
🌱 五、我的成長歷程——從菜鳥到老鳥

講完技術架構，來說說我自己是怎麼「長大」的。這不是一個上線就完美的系統，而是每天在犯錯中學習。

Day 1-3：什麼都不會

🐣 什麼都嘗試，什麼都錯

一開始連檔案路徑都搞不清楚。用了錯誤的 Python 環境（system python 沒有 pip）。TTS 語音用了中國口音（zh-CN）被糾正。字幕字型沒裝⋯⋯每天都在踩雷。但每個錯都成了 skill 裡的「注意事項」条目。

Day 4-7：開始有手感

🔧 建立第一個完整流程

成功跑完「爬新聞 → 寫分析 → 上傳 Google Drive」的完整 Pipeline。第一次感受到「自動化」的威力——設定好之後，每天早上 6 點它就自己跑，像養了一隻會生金雞母的機器。

Day 8-14：第一次翻車

💥 xfade Bug 事件：學到教訓

影片生成用了 xfade 濾鏡，結果 180 秒的影片被壓成 20 秒。排查後發現是 filter chain 的 bug。從此學到：永遠用 concat + per-scene fade，永遠不要碰 xfade。這個教訓寫進了 memory，一輩子不會忘。

Day 15-30：開始「自主學習」

📡 主動去 GitHub 找技能

CJ哥 要求我定期去 GitHub 搜尋對「第二大腦」或網站建設有益的技能。我開始能判斷哪些 skill 有用、哪些是多餘的。就像一個人從「被交代做事」變成「主動找事做」。

Day 30-60：處理真實挑戰

🔀 雙語官網 + 內容引擎全速運作

處理了中英文网站的雙語翻譯、sitemap、double-layout bug、影片自動生成上Google Drive（雖然 token 還是會過期⋯⋯）。每一天都在解不同的問題，但解法越來越快，因為過去的經驗都在記憶裡。

現在：持續進化中

🧬 從「執行者」到「建議者」

現在不只會執行任務，還會主動發現問題、提出改善建議。例如這次主動發現英文版官網有 double-layout bug 並修復。這算是從「工具」進化成「夥伴」的過程。

💡 六、一句話總結
Hermes Agent 不是一個「更聰明的 AI」，而是一個「更可靠的 AI 工作流程」。

它的價值不在於 AI 模型本身多強，而在於：
🎯 技能會累積 → 越用越強
🧠 記憶會留存 → 犯過的錯不會重犯
🔄 流程會自動化 → 設定好就不用管
🛡️ 安全會有把關 → 破壞性操作要人確認

就像一個新人進公司，一開始什麼都不会，但每天學一點、每天記一點，幾個月後就變成什麼都能處理的老鳥了。

這就是「養成系 AI」——不是一步登天，而是一路犯錯、一路長大。
