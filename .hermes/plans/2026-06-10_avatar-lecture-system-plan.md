## 二、Avatar 系統需要的核心能力

### 2.1 數位人臉生成（Avatar Face）

**方案 A：HeyGen API**（推薦）
- 照片/影片 → 生成會說話的 Avatar
- 支援唇形同步、多語言
- API 呼叫簡單，品質高
- 費用：約 $24/月（Creator 方案）
- 需要：API Key + 頭像照片

**方案 B：D-ID API**
- 靜態照片 + 音訊 → 說話影片
- 支援手勢、表情
- 費用：約 $5.9/月（Starter）
- 需要：API Key + 頭像照片

**方案 C：SadTalker / Wav2Lip**（開源）
- 本地執行，免費
- 需要 GPU（至少 8GB VRAM）
- 品質較差，但可自訂
- 需要：Python + PyTorch + 模型權重

**方案 D：HeyGen Streaming API**
- 即時串流 Avatar
- 適合直播/互動課程
- 費用較高

**建議：先用 D-ID（低成本驗證），品質不夠再升級 HeyGen**

### 2.2 唇形同步（Lip Sync）

| 方案 | 說明 | 成本 |
|------|------|------|
| D-ID 內建 | 音訊驅動唇形 | 包含在 D-ID |
| SadTalker | 開源，本地跑 | 免費（需 GPU）|
| Wav2Lip | 開源，精準度高 | 免費（需 GPU）|
| HeyGen 內建 | 高品質唇形 | 包含在 HeyGen |

### 2.3 手勢與身體語言

| 方案 | 說明 |
|------|------|
| HeyGen 內建 | 自動手勢（有限）|
| SadTalker + 姿勢估計 | 開源，可自訂 |
| 手動標註 | 在腳本中指定手勢時間點 |
| 無手態 | 只有頭像 + 唇形（最簡單）|

**建議：初期先做「頭像 + 唇形」，手勢後面再加**

### 2.4 場景與背景

| 方案 | 說明 |
|------|------|
| 靜態背景圖 | 最簡單，Pillow 生成 |
| 動態背景影片 | ffmpeg 疊加 |
| 虛擬攝影棚 | HeyGen/D-ID 內建 |
| 3D 場景 | Unity/Unreal（過度工程）|

**建議：靜態背景 + B-roll 疊加（現有管線延伸）**

## 七、技術規格建議

### 影片格式
- 解析度：1920×1080（橫向）或 1080×1920（直向/Shorts）
- 幀率：30fps
- 編碼：H.264 + AAC
- 時長：5-10 分鐘

### Avatar 規格
- 頭像：正面、光線均勻、背景單純
- 解析度：至少 512×512
- 格式：JPG/PNG
- 建議：專業形象照或 AI 生成頭像

### 語音規格
- 模型：ElevenLabs v3（已有）
- 語速：105-110%（偏快但有精神）
- 停頓：句號 250ms、逗號 80ms
- 強調：數字、關鍵詞自動標註
